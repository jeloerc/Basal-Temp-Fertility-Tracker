Basal Temperature Tracker Mobile App Guide

Getting Started

Welcome to the Basal Temperature Tracker mobile application guide. This app helps you track and visualize your basal body temperature throughout your menstrual cycle.

  • Make sure the backend server is running at http://192.168.50.164:5000
  • Download the Expo Go app on your mobile device from the App Store or Google Play
  • Navigate to the react-native-app directory on the server and run ./start.sh
  • Scan the QR code that appears with your Expo Go app
  • The app should connect to your data automatically
    • Note: Your mobile device must be on the same network as the server for the connection to work properly.

    Understanding the Interface

    Main Screen

    The main screen displays your temperature chart and cycle information:

    • Top Bar: Shows cycle selector
    • Cycle Info Card: Displays start/end dates, length, average temperature
    • Temperature Chart: Visual representation of your basal temperatures
    • Stats Section: Shows cycle analysis data

    Temperature Chart

    The chart uses color coding to help you understand your cycle:

    • Purple Line: Follicular phase (pre-ovulation)
    • Pink Line: Luteal phase (post-ovulation)
    • Red Markers: Period days
    • Green Line: Fertile threshold
    • Lime Green Area: Potentially fertile zone

    Features

    Cycle Navigation

    Tap on cycle buttons at the top to switch between different cycles.

    Temperature Data

    Tap on any point in the chart to see detailed information about that day.

    Temperature Unit

    Use the button at the bottom of the screen to switch between Fahrenheit and Celsius.

    Fertility Analysis

    The app automatically analyzes your temperature patterns to identify:

    • Temperature shifts indicating potential ovulation
    • Follicular and luteal phases
    • Cycle statistics to help identify patterns

    Troubleshooting

    Connection Issues

    • Verify that the Flask backend is running
    • Make sure your device is on the same network as the server
    • Check that the server IP is correctly set in App.js
    • Try restarting the Expo server and reload the app

    Data Not Showing

    • Verify that you have data entered in the web application
    • Check the API endpoint is working by visiting http://192.168.50.164:5000/api/cycle_analytics in a browser
    • Use the Retry button in the app if loading fails
